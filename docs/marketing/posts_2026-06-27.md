## Post 1 — r/productivity

**Titre** :
```
[Tool] I got tired of losing my streak record every time I broke a streak, so I built a local-first tracker that remembers your all-time best
```

**Body** (markdown) :

```
## The problem
Every habit tracker I tried had the same flaw: when I broke a streak (travel, sickness, life), my "best streak" record was either reset forever or never tracked in the first place. Habitica's own wiki literally says: "After a streak counter has been reset to zero, there is no way to find out what the streak used to be."

I wanted to see three things at a glance:
- **Current streak** (so I know if I should keep going today)
- **Best streak ever** (so I have something to beat, even if my current run is at 0)
- **Longest gap** (so I know what to plan around)

## What I built
A 100% local, Windows desktop habit tracker. ~3 MB installer, no cloud, no telemetry, no account. Your data never leaves your machine.

The interesting bit for this subreddit: every habit now has a **persistent** `bestStreak` and `longestGap` field that's recalculated after every check-in and survives a reset. The CSV export now gives you lifetime stats per row (current_streak, best_streak, completion_rate_30d, total_completed) so you can actually analyze your patterns in Excel/Notion.

There's also a GitHub-style 365-day heatmap and a 30-day rolling sparkline per habit, in case you like that kind of thing.

## Stack (for the curious)
- Tauri v2 (Rust + WebView2)
- React 19 + TypeScript
- Pure CSS, no UI lib
- localStorage with FNV-1a checksum + backup key for integrity
- 171 tests, 0 cloud calls

## What I'm NOT claiming
- It's not gamified (no avatars, no XP)
- It's Windows-only (Tauri WebView2)
- It's not mobile

## Link
https://github.com/Lemniscate-world/LifeTrack/releases/tag/v0.1.1

Would love to hear:
- What stats do YOU look at most when reviewing your habits?
- Is "all-time best" actually useful to you, or does it just remind you of failures?
- Anyone else tracking "longest gap" as a forward-planning metric?
```

Hey everyone,

I've been doing the X-effect method for a few habits and tracking them in a small Windows app I built (Tauri + React, ~3 MB installer, all local).

What changed in v0.1.1:
- **Best streak ever** is now persisted on the habit itself, so a 47-day streak I lost in March is still visible today. The current run is at 0, but my record is 47.
- **Longest gap** — answers "what's the longest I've ever gone without this?" which is super useful for planning breaks.
- **GitHub-style heatmap** per habit — 365 days, one cell per day. Makes the year visible at a glance.

Curious what the X-effect community thinks — for those of you using pen & paper or spreadsheets, would these visualizations actually help, or is part of the X method that you DON'T visualize and just trust the chain?

### PyTorch / Side-project Discord (en anglais, format "show and tell")

```
Built a habit tracker where the most interesting part is the stats layer, not the UI.

Every habit has a persistent `bestStreak` and `longestGap` field that's recomputed after every check-in. The 365-day heatmap is GitHub-style SVG (53×7 grid, ~400 rects per habit) — no d3, no recharts.

The fun engineering bit: the streak calculation needs to handle
- backdated habit creation (created today, checked yesterday)
- tracking boundary (no penalty for pre-existence)
- same-day creation (don't count today as a missed day)

~400 lines of pure TS in `src/stats.ts`, 29 unit tests covering all the edge cases. The whole thing is local-first (Tauri + WebView2, 3 MB installer).

If anyone has thoughts on the streak math (especially the boundary cases), would love a review:
https://github.com/Lemniscate-world/LifeTrack/blob/main/src/stats.ts
```
